Cryptography

  • Caesar cipher: substitution only
  • more general monoalphabetic substitution
  • Vigener (polyalphabetic)
  • transposition cipher
  • combining the two
  • public key crypto
  • Diffie Hellman based on modular arithmetic 
          p = 2**31 - 1 = 2147483647
      base = 1023
      suppose we want to solve base**x = 1000
      what about base**x % p = 1000

      what facts do you know about (base ** x) ** y?
    p must be prime, the base must be a generator Do some examples in Python.
  • If p is not prime or base is not a generator, then base ** x will repeat for multiple x's making it easier to brute-force
  • How Diffie-Hellman works: 
          (a ** b) ** c
  • RSA math is more complex. There are 2 large primes n = p * q 
          a ** (k * (p-1)*(q-1) + 1) % n = a
      e * d = k * (p-1) * (q-1) + 1
      e is public
      d is private
  • You odn't need to brute force RSA if you can factor n.
  • Some functions have inverses and some don't
  • A cipher needs an inverse, a hash does not have one
  • public key: the inverse cannot be derived from the cipher
